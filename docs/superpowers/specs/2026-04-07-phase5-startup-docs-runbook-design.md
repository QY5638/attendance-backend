# Phase 5 启动文档与联调手册首包设计

## 背景

`Phase 4 - 智能链路治理完善` 已完成，当前进入 `Phase 5 - 文档、部署、联调、验收闭环`。现状最明显的交付缺口不是代码，而是前后端仓库的启动文档几乎为空：

- `backend/README.md` 只有仓库标题。
- `frontend/README.md` 只有仓库标题。
- 现有示例配置与 CI 信息分散在多个文件中，没有一份可以指导新机器或新会话从零启动并完成联调的稳定文档。

因此，Phase 5 第一轮先锁定一个最小且独立的文档任务包：`启动文档 + 独立联调文档`。

## 目标

交付三份可直接使用的文档，使新机器或新会话至少能完成以下事情：

1. 仅阅读后端仓库 README，即可知道后端依赖、配置、启动、测试和最小排障方法。
2. 仅阅读前端仓库 README，即可知道前端依赖、环境变量、开发启动、构建与测试方法。
3. 阅读一份独立联调手册，即可按顺序完成“后端启动 -> 前端启动 -> 最小主链联调”的串联操作。

本轮不要求完成最终验收清单、截图模板或一键脚本。

## 范围

### 本轮包含

- 完整重写 `backend/README.md`
- 完整重写 `frontend/README.md`
- 新增一份独立联调文档，建议路径：`docs/integration/phase5-local-runbook.md`
- 回写 `D:\Graduation project\继续执行文档-完整系统.md` 的阶段状态与本轮记录

### 本轮不包含

- 新增初始化脚本、部署脚本或 Docker 文件
- 修改后端 CI 或前端工程配置
- 修改业务代码或接口行为
- 编写最终验收步骤、截图规范、证据模板

## 文件职责

### 1. `backend/README.md`

职责：只说明后端单仓库怎么跑。

计划覆盖：

- 项目简介与技术栈
- 环境要求：JDK、Maven、MySQL、Redis
- 配置文件：`application-local.yml` 与 `application-local.example.yml` 的关系与必填项
- 启动步骤：数据库准备、启动命令、端口说明
- 测试与编译命令：对齐 `pom.xml` 与后端 CI
- 常见问题：数据库连接、Redis、LLM、地图配置缺失时的典型现象

### 2. `frontend/README.md`

职责：只说明前端单仓库怎么跑。

计划覆盖：

- 项目简介与技术栈
- 环境要求：Node、npm
- 环境变量：`.env.local` 与 `.env.example` 的关系
- 启动/构建/测试命令：对齐 `package.json`
- 与后端代理的关系：`VITE_API_PROXY_TARGET`
- 常见问题：代理失败、登录失败、地图 key 缺失

### 3. `docs/integration/phase5-local-runbook.md`

职责：专门说明跨仓库的联调顺序，不与单仓库 README 混写。

计划覆盖：

- 适用范围与前置依赖
- 推荐启动顺序：后端配置 -> 后端启动 -> 前端配置 -> 前端启动
- 最小联调路径：登录 -> 打卡/异常 -> 预警 -> 复核 -> 统计
- 每一步的入口页面、前置条件与预期现象
- 最小验证命令与人工验证动作
- 常见联调阻塞的排查顺序

### 4. `继续执行文档-完整系统.md`

职责：维持续跑状态，不承载详细操作手册。

本轮只回写：

- `Phase 5` 已启动
- 本轮完成的是“启动文档 + 联调手册首包”
- 本轮未完成内容仍是“最终验收清单/证据要求”
- 下一步第一动作与最小验证结果

## 内容边界与写法原则

- 文档必须全部使用中文。
- 不写真实密钥、账号密码或私有环境值。
- 不虚构“已自动化”的能力；如果当前仍需手工准备 MySQL/Redis/LLM/地图配置，就明确写成手工步骤。
- README 负责“单仓库如何启动”，联调手册负责“两个仓库如何串起来”。
- 不把 README 写成冗长设计文档；只保留新机器启动真正需要的信息。

## 验证策略

本轮是文档交付，验证以“可执行性校对”为主，不新增业务代码测试：

1. 校对 README 中引用的命令、路径、配置文件名是否在仓库真实存在。
2. 校对后端命令与 `backend/.github/workflows/backend-ci.yml`、`backend/pom.xml` 一致。
3. 校对前端命令与 `frontend/package.json`、`frontend/.env.example` 一致。
4. 校对联调文档中的端口、代理目标、配置路径与当前仓库现状一致。
5. 回写 `继续执行文档-完整系统.md`，记录本轮完成、未完成、下一步与验证结果。

## 风险与后续

- 本轮无法凭文档本身解决外部依赖缺失问题，只能把依赖准备说清楚。
- 由于当前缺少一键初始化脚本，文档会保留必要的手工准备步骤。
- `Phase 5` 在本轮之后仍未整体完成；下一轮应继续补“最终验收步骤、证据要求与联调闭环输出”。

## 隔离策略

- 本设计文档写入后端隔离 worktree：`backend/.worktrees/phase5-docs-runbook`
- 后续若实际修改 `frontend/README.md`，实现阶段需先为前端仓库创建独立非 `main` worktree，再执行文档修改
- 不直接在 `backend/main` 或 `frontend/main` 上开发
