# 用例图与流程图文字说明

## 1. 系统用例图文字说明

### 1.1 参与者
系统主要包含两个参与者：
- 管理员
- 员工

### 1.2 员工用例
- 用户登录
- 人脸信息录入
- 发起考勤打卡
- 查看个人考勤记录
- 查看个人异常记录
- 提交补卡申请
- 查看补卡处理结果

### 1.3 管理员用例
- 用户登录
- 管理员工信息
- 管理部门信息
- 配置考勤规则
- 查看考勤记录
- 查看异常记录
- 查看预警信息
- 进行人工复核
- 查看统计报表
- 导出报表数据

### 1.4 核心用例关系说明
- “考勤打卡”包含“人脸识别”
- “异常检测”包含“规则判定”
- “复杂异常检测”扩展自“异常检测”
- “人工复核”扩展自“复杂异常检测”
- “预警推送”包含于“异常结果生成”

## 2. 业务流程图文字说明

### 2.1 员工正常打卡流程
1. 员工登录系统
2. 员工进入打卡页面
3. 系统调用摄像头采集当前人脸图像
4. 系统执行人脸检测与身份匹配
5. 若匹配成功，则生成打卡记录
6. 规则引擎对打卡记录进行基础异常判定
7. 若无异常，则将打卡结果标记为正常并写入数据库
8. 系统向员工返回打卡成功提示

### 2.2 基础异常检测流程
1. 系统接收到打卡记录
2. 调用考勤规则进行时间校验
3. 判断是否存在迟到、早退、缺勤、重复打卡等情况
4. 若命中基础异常，则直接生成异常记录
5. 系统保存异常记录并推送预警信息
6. 管理员可在后台查看异常详情

### 2.3 复杂异常检测流程
1. 员工完成打卡后生成原始记录
2. 系统进行规则引擎初步判定
3. 若发现设备异常、地点异常、人脸分数异常、行为模式异常等风险特征，则进入复杂异常分析流程
4. 系统提取多维风险特征，包括身份、时间、设备、空间和历史行为等信息
5. 系统构造结构化提示词并调用大模型 API
6. 大模型输出异常类型、风险等级、判定依据和处理建议
7. 系统融合规则结果与模型结果生成最终判定
8. 若风险等级较高，则生成预警并推送管理员
9. 若结果不确定，则转入人工复核流程

### 2.4 人工复核流程
1. 管理员接收到异常预警
2. 进入异常详情页面查看判定信息
3. 查看规则判定结果、大模型分析依据和历史记录
4. 管理员进行确认、驳回或标记误报处理
5. 系统保存复核结果并更新异常处理状态
6. 复核结果用于后续优化异常判定策略

### 2.5 统计分析流程
1. 系统定期汇总考勤记录、异常记录和复核记录
2. 按员工、部门、时间维度进行分类统计
3. 生成出勤率、异常率、趋势分析等统计结果
4. 在前端页面展示图表与报表
5. 支持导出 Excel 或 PDF 报表
