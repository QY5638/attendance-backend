# 考勤异常检测与预警系统模块划分说明书

## 1. 文档说明

### 1.1 编写目的
本文档用于说明考勤异常检测与预警系统的模块划分原则、模块结构、职责边界、模块间关系与数据流向，为系统设计、实现和后续维护提供依据。

### 1.2 划分目标
- 使系统结构清晰、职责明确
- 提高模块复用性与可维护性
- 便于前后端分工和后续功能扩展
- 突出复杂异常检测模块的核心创新价值

## 2. 模块划分原则
- 单一职责原则：每个模块只负责一类核心业务
- 高内聚低耦合原则：模块内部功能紧密相关，模块间依赖最小化
- 分层设计原则：表现层、业务层、数据层、智能分析层分离
- 可扩展原则：为新增异常场景和规则扩展预留空间
- 安全性原则：敏感数据处理与权限控制独立管理

## 3. 系统总体模块结构
系统总体划分为以下九个核心模块：
- 用户与权限管理模块
- 基础资料管理模块
- 人脸采集与识别模块
- 考勤记录管理模块
- 基础异常规则判定模块
- 复杂异常智能分析模块
- 风险预警与人工复核模块
- 统计分析与报表模块
- 系统支撑模块

## 4. 模块详细划分

### 4.1 用户与权限管理模块
职责：负责系统用户身份管理与访问控制，包括员工账号、管理员账号、角色权限分配和登录认证。

子功能：
- 用户登录
- 用户信息管理
- 角色管理
- 权限校验
- 账号状态控制

### 4.2 基础资料管理模块
职责：负责维护系统运行所需的基础数据，是其他业务模块的支撑模块。

子功能：
- 部门信息管理
- 考勤规则管理
- 打卡设备信息管理
- 风险等级配置
- 异常类型配置

### 4.3 人脸采集与识别模块
职责：负责员工人脸信息采集、特征提取和身份识别，是打卡流程的前置模块。

子功能：
- 摄像头采集
- 人脸检测
- 活体校验
- 特征提取
- 特征匹配

### 4.4 考勤记录管理模块
职责：负责打卡记录的生成、存储、查询和状态维护，是系统业务主线模块。

子功能：
- 上班打卡
- 下班打卡
- 补卡申请
- 打卡记录查询
- 打卡状态维护

### 4.5 基础异常规则判定模块
职责：负责对规则明确的考勤异常进行快速判定，是异常识别的第一层。

子功能：
- 迟到判定
- 早退判定
- 缺勤判定
- 重复打卡判定
- 非法时间打卡判定

### 4.6 复杂异常智能分析模块
职责：负责对规则难以准确覆盖的复杂异常场景进行分析，是系统的核心创新模块。

子功能：
- 风险特征提取
- 大模型提示词构造
- 代打卡分析
- 多设备异常分析
- 多地点异常分析
- 行为组合异常分析
- 结果融合与风险评分

### 4.7 风险预警与人工复核模块
职责：负责对异常结果进行预警推送、状态追踪和人工复核处理。

子功能：
- 预警生成
- 风险分级
- 预警列表展示
- 异常详情查看
- 管理员复核
- 复核结果归档

### 4.8 统计分析与报表模块
职责：负责对考勤、异常、预警和复核数据进行统计汇总与可视化展示。

子功能：
- 个人考勤统计
- 部门考勤统计
- 出勤率分析
- 异常率分析
- 趋势图展示
- 报表导出

### 4.9 系统支撑模块
职责：负责系统通用功能和底层支撑能力，为其他模块提供公共服务。

子功能：
- 日志管理
- 异常处理
- 文件处理
- 缓存管理
- 数据加密
- 模型 API 调用封装
- 定时任务
- 通用工具类

## 5. 模块关系说明
系统主业务链路如下：

用户与权限管理模块 -> 人脸采集与识别模块 -> 考勤记录管理模块 -> 基础异常规则判定模块 -> 复杂异常智能分析模块 -> 风险预警与人工复核模块 -> 统计分析与报表模块

### 5.1 模块与数据库表对应关系

系统数据统一落库到 `system` 数据库，核心模块与主要数据表的对应关系如下：

| 模块 | 主要数据表 |
|---|---|
| 用户与权限管理模块 | `user`、`department`、`role` |
| 人脸采集与识别模块 | `faceFeature` |
| 考勤记录管理模块 | `attendanceRecord`、`device` |
| 基础异常规则判定模块 | `rule`、`attendanceException` |
| 复杂异常智能分析模块 | `exceptionAnalysis` |
| 风险预警与人工复核模块 | `warningRecord`、`reviewRecord` |
| 系统支撑模块 | `operationLog` |

数据库表名采用简化命名风格，如 `attendanceRecord`、`warningRecord`；字段名采用小驼峰风格，如 `deptId`、`roleId`、`createTime`。

## 9. 模块划分优势
- 结构清晰，便于论文展示和答辩讲解
- 职责明确，便于代码实现与团队分工
- 规则引擎与大模型模块独立，便于突出创新点
- 后续可扩展更多异常类型和分析策略
