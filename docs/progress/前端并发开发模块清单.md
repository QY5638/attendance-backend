# 前端并发开发模块清单

## 1. 文档说明

本文档用于固定前端可并发开发的页面模块、后端接口依赖、前置前端模块和完成标准，供后续其他会话直接读取并继续并发开发。

适用目录：`D:\Graduation project\frontend`

默认参考资料：
- `D:\Graduation project\frontend\AGENTS.md`
- `D:\Graduation project\backend\docs\api\API接口设计文档.md`
- `D:\Graduation project\backend\docs\architecture\模块划分说明书.md`

## 2. 共享前提

- 当前前端仍是骨架仓库，公共壳层、路由、状态管理、页面目录可按模块并发搭建。
- 所有页面模块默认依赖 `FE-01 公共壳层与认证模块` 提供登录态、角色菜单和受保护路由。
- 后端接口契约以 `API接口设计文档.md` 为准；如接口尚未实现，页面可先按契约完成结构与交互骨架。

## 3. 模块清单

### FE-01 公共壳层与认证模块

- 模块名：公共壳层与认证模块
- 页面/路由范围：`/login` 与后台主壳层
- 依赖后端接口组：
  - `POST /api/auth/login`
- 依赖前置前端模块：无
- 可并行边界：只负责登录、Token/JWT、角色识别、菜单与路由守卫；不负责具体业务页面。
- 完成标准：登录成功后可进入后台；未登录访问后台页被拦截；能基于 `roleCode` 控制页面可见性。

### FE-02 概览与统计模块

- 模块名：概览与统计模块
- 页面/路由范围：`/dashboard`、`/statistics`
- 依赖后端接口组：
  - `GET /api/statistics/personal`
  - `GET /api/statistics/department`
  - `GET /api/statistics/exception-trend`
  - `GET /api/statistics/summary`
  - `GET /api/statistics/department-risk-brief`
  - `GET /api/statistics/export`
  - `GET /api/warning/list`
- 依赖前置前端模块：`FE-01`
- 可并行边界：负责首页概览、趋势图、AI 统计摘要、风险画像、报表与导出入口；不处理复核、用户维护、系统配置。
- 完成标准：`dashboard` 可展示概览、预警摘要和 AI 摘要；`statistics` 可展示统计、趋势、风险画像并触发导出。

### FE-03 用户、部门与角色模块

- 模块名：用户、部门与角色模块
- 页面/路由范围：`/user`、`/department`、`/system/role`
- 依赖后端接口组：
  - `GET /api/user/list`
  - `POST /api/user/add`
  - `PUT /api/user/update`
  - `DELETE /api/user/{id}`
  - `GET /api/department/list`
  - `POST /api/department/add`
  - `PUT /api/department/update`
  - `DELETE /api/department/{id}`
  - `GET /api/role/list`
  - `POST /api/role/add`
  - `PUT /api/role/update`
  - `DELETE /api/role/{id}`
- 依赖前置前端模块：`FE-01`
- 可并行边界：只负责员工、部门、角色资料页面；不承担登录、人脸、打卡、异常处理。
- 完成标准：用户、部门、角色列表与表单可闭环；用户表单可选择部门与角色；角色配置可独立维护。

### FE-04 人脸采集模块

- 模块名：人脸采集模块
- 页面/路由范围：`/face`
- 依赖后端接口组：
  - `POST /api/face/register`
  - `POST /api/face/verify`
- 依赖前置前端模块：`FE-01`
- 冻结契约补充：`POST /api/face/register` 与 `POST /api/face/verify` 均为当前登录用户自助接口，请求体可不传 `userId`；若携带 `userId`，后端会忽略并以当前会话用户覆盖。
- 冻结契约补充：上述两个人脸接口均对 `ADMIN`、`EMPLOYEE` 开放。
- 冻结契约补充：两个人脸接口响应中的 `userId` 表示当前会话解析出的用户编号。
- 可并行边界：只负责人脸录入、重录和结果反馈；不负责打卡和异常分析。
- 完成标准：可完成当前用户人脸录入；有成功/失败反馈；可独立联调。

### FE-05 考勤打卡与个人记录模块

- 模块名：考勤打卡与个人记录模块
- 页面/路由范围：`/attendance`
- 依赖后端接口组：
  - `POST /api/attendance/checkin`
  - `GET /api/attendance/device-options`
  - `GET /api/attendance/record/me`
  - `POST /api/attendance/repair`
  - `POST /api/face/verify`（当前登录用户自助；请求体若携带 `userId` 会被后端忽略）
- 依赖前置前端模块：`FE-01`
- 冻结契约补充：`POST /api/attendance/checkin` 与 `POST /api/attendance/repair` 为当前登录用户自助接口，请求体可不传 `userId`；若携带 `userId`，后端会忽略并以当前会话用户覆盖。
- 冻结契约补充：`GET /api/attendance/device-options` 是 `FE-05` 唯一设备选择来源，不直接复用 `/api/device/list`。
- 冻结契约补充：`GET /api/attendance/record/me` 是 `FE-05` 唯一自助入口；`GET /api/attendance/record/{userId}` 仅保留为兼容/管理接口，不作为 `FE-05` 入口。
- 冻结契约补充：`FE-05` 不依赖 `FE-01` 登录响应补充数值型 `userId`，也不要求改造前端 auth store。
- 冻结契约补充：`POST /api/attendance/checkin` 只需前端传 `checkType`、`deviceId`、`imageData`；`ipAddr` 由后端按 request 地址解析，`location` 由后端按设备位置快照回填。
- 可并行边界：只负责打卡提交、个人记录、补卡申请与结果反馈；不处理异常列表、预警和复核。
- 完成标准：可提交上/下班打卡；可查看个人记录；可提交补卡申请；结果反馈清晰。

### FE-06 异常与预警模块

- 模块名：异常与预警模块
- 页面/路由范围：`/exception`、`/warning`
- 依赖后端接口组：
  - `GET /api/exception/list`
  - `GET /api/exception/{id}`
  - `GET /api/exception/{id}/decision-trace`
  - `GET /api/exception/{id}/analysis-brief`
  - `GET /api/warning/list`
  - `GET /api/warning/{id}/advice`
- 依赖前置前端模块：`FE-01`
- 可并行边界：负责异常列表、异常详情、预警列表、AI 判定依据与决策链展示；不负责提交复核结果。
- 完成标准：异常列表可查；异常详情可看 AI 摘要和决策链；预警列表可展示并跳转关联异常。

### FE-07 人工复核模块

- 模块名：人工复核模块
- 页面/路由范围：`/review`
- 依赖后端接口组：
  - `POST /api/review/submit`
  - `GET /api/review/{exceptionId}`
  - `GET /api/review/{exceptionId}/assistant`
  - `POST /api/review/feedback`
  - `GET /api/exception/{id}`
- 依赖前置前端模块：`FE-01`
- 可并行边界：负责复核详情、确认/驳回、复核历史、AI 复核建议与反馈标签；不负责异常列表和预警汇总页。
- 完成标准：可基于异常 ID 查看详情；可查看 AI 复核建议；可提交复核结果和反馈标签；可查看复核历史。

### FE-08 系统配置模块

- 模块名：系统配置模块
- 页面/路由范围：`/system`
- 依赖后端接口组：
  - `GET /api/device/list`
  - `POST /api/device/add`
  - `PUT /api/device/update`
  - `PUT /api/device/status`
  - `DELETE /api/device/{deviceId}`
  - `GET /api/rule/list`
  - `POST /api/rule/add`
  - `PUT /api/rule/update`
  - `PUT /api/rule/status`
  - `GET /api/system/prompt/list`
  - `POST /api/system/prompt/add`
  - `PUT /api/system/prompt/update`
  - `PUT /api/system/prompt/status`
  - `GET /api/system/risk-level/list`
  - `PUT /api/system/risk-level/update`
  - `GET /api/system/exception-type/list`
  - `PUT /api/system/exception-type/update`
  - `GET /api/log/operation/list`
  - `GET /api/system/model-log/list`
- 依赖前置前端模块：`FE-01`
- 可并行边界：负责规则、设备、提示词模板、风险等级、异常类型、模型调用日志等配置页；其中操作日志接口由后端统计与审计查询能力提供；不负责用户资料、打卡、异常处置和统计报表。
- 完成标准：系统配置页入口齐全；规则、设备、提示词模板、风险等级、异常类型、日志查询可形成页面闭环。

## 4. 必须先落地的模块

- `FE-01 公共壳层与认证模块`

原因：其他页面都依赖登录态、菜单、角色识别和受保护路由。

## 5. 可独立并发开发的模块

在 `FE-01` 完成后，可独立并发：
- `FE-02 概览与统计模块`
- `FE-03 用户、部门与角色模块`
- `FE-04 人脸采集模块`
- `FE-05 考勤打卡与个人记录模块`
- `FE-06 异常与预警模块`
- `FE-08 系统配置模块`

`FE-07 人工复核模块` 可并发，但要先冻结异常详情字段与 `exceptionId` 跳转契约。

## 6. 推荐并发波次

- 第一波：`FE-01`
- 第二波并发：`FE-02 + FE-03 + FE-04 + FE-05 + FE-06 + FE-08`
- 第三波：`FE-07`

## 7. 接口缺口提醒

- 本轮已在 API 主文档中补齐部门、角色、设备、考勤记录、补卡、系统配置相关接口。
- 但“已写入文档”不等于“后端已实现”，联调前需先核对后端模块实际开发进度。
- 前端实现时，若发现文档字段与后端返回不一致，应优先回写 API 文档，再统一调整页面实现。

## 8. 使用说明

- 新会话开始后，先读取本文件，再读取 `frontend/AGENTS.md` 与后端 API 文档。
- 每次只认领一个前端模块编号，减少多人同时改同一页面组的冲突。
- 如需补路由、菜单、接口封装，请优先保证不跨模块侵入。
- 若接口契约发生变化，必须同步更新本文件与 API 主文档。
