# 前后端模块开发与联调对应关系

## 说明

- 前端页面开发可以先按 API 契约完成页面结构、交互骨架和假数据联调，不必等待后端全部实现。
- 但真实联调必须以对应后端模块已经落地为前提。
- 除 `FE-01` 外，其余前端模块默认都依赖 `FE-01` 提供登录态、角色菜单和受保护路由。

## 总览表

| 前端模块 | 页面范围 | 页面直连的后端模块 | 完整联调前提（后端链路） | 说明 |
| --- | --- | --- | --- | --- |
| `FE-01 公共壳层与认证` | `/login`、后台主壳层 | `BE-01` | `BE-01` | 只依赖登录、Token、角色识别能力 |
| `FE-02 概览与统计` | `/dashboard`、`/statistics` | `BE-06 + BE-08` | `BE-04 + BE-05 + BE-06 + BE-07 + BE-08` | 页面直接用预警和统计接口，但统计结果本质依赖完整业务链路 |
| `FE-03 用户/部门/角色` | `/user`、`/department`、`/system/role` | `BE-01` | `BE-01` | 对应用户、部门、角色 CRUD |
| `FE-04 人脸采集` | `/face` | `BE-03` | `BE-01 + BE-03` | 人脸录入/验证要建立在员工主体存在的前提上 |
| `FE-05 考勤打卡与个人记录` | `/attendance` | `BE-03 + BE-04` | `BE-01 + BE-02 + BE-03 + BE-04` | 打卡页既依赖人脸验证，也依赖考勤记录落库 |
| `FE-06 异常与预警` | `/exception`、`/warning` | `BE-05 + BE-06` | `BE-04 + BE-05 + BE-06` | 异常页面依赖异常分析，预警页面依赖预警生成 |
| `FE-07 人工复核` | `/review` | `BE-05 + BE-07` | `BE-04 + BE-05 + BE-07` | 复核页需要先有异常详情，再有复核处理能力 |
| `FE-08 系统配置` | `/system` | `BE-02 + BE-05 + BE-06 + BE-07 + BE-08` | 同左，另含智能分析支撑能力 | 系统配置页是跨后端模块最多的页面组 |

## FE-08 系统配置模块拆解

- 设备配置页：依赖 `BE-02`
- 规则配置页：依赖 `BE-05`
- 提示词模板管理：当前应依赖 `BE-05` 相关智能分析支撑能力
- 模型调用日志查询：当前应依赖 `BE-05` 相关智能分析支撑能力
- 风险等级配置：依赖 `BE-06`
- 异常类型配置：依赖 `BE-07`
- 操作日志查询：依赖 `BE-08`

## 哪些前端可以先开发

- 所有前端模块都可以先按接口文档做页面骨架、交互流程和前端状态管理。
- 但只有对应后端模块完成后，才能进入真实联调。
- `FE-07` 还需要先冻结异常详情字段和 `exceptionId` 跳转契约，再做稳定联调。

## 建议联调顺序

1. `FE-01 <-> BE-01`
2. `FE-03 <-> BE-01`
3. `FE-04 <-> BE-03`
4. `FE-05 <-> BE-03 + BE-04`
5. `FE-06 <-> BE-05`，再接 `BE-06`
6. `FE-07 <-> BE-05 + BE-07`
7. `FE-08` 按子页分批联调：设备先对 `BE-02`；规则、提示词、模型日志再对 `BE-05`；风险等级对 `BE-06`；异常类型对 `BE-07`；操作日志对 `BE-08`
8. `FE-02` 放在 `BE-08` 完成后，再和 `BE-06` 一起做首页概览与统计联调

## 按当前后端进度的实际情况

- `BE-01`、`BE-02` 已完成并合并
- 当前可真实联调的前端部分主要是：`FE-01`、`FE-03`、`FE-08` 中的设备配置部分
- `FE-04` 及之后模块可以先开发页面骨架，但真实联调要等对应后端模块完成
