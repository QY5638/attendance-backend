# 后端并发开发模块清单

## 1. 文档说明

本文档用于固定后端可并发开发的模块边界、前置依赖、接口组和完成标准，供后续其他会话直接读取并继续并发开发。

适用目录：`D:\Graduation project\backend`

默认参考资料：
- `D:\Graduation project\backend\AGENTS.md`
- `D:\Graduation project\backend\docs\architecture\模块划分说明书.md`
- `D:\Graduation project\backend\docs\api\API接口设计文档.md`
- `D:\Graduation project\backend\docs\database\数据库设计文档.md`

## 2. 共享前提

- 已有公共基础层：`common`、`config`、统一返回、全局异常处理、基础安全配置、健康检查骨架。
- 数据库与命名约定已固定：数据库名 `system`，表名简化，字段小驼峰，中文注释。
- 后续所有业务开发都默认围绕现有包结构推进：`auth/user/department/role/device/face/attendance/exceptiondetect/warning/review/statistics`。
- `device` 表主键字段当前为 `id`，但接口契约统一对外暴露为 `deviceId`，后续实现需保持该映射关系。
- `rule` 视为 `exceptiondetect` 的规则子域，不额外拆独立顶层包。
- 系统支撑层需补充模型中台能力，包括提示词模板管理、模型调用日志、结构化输出解析与决策追踪。
- `/api/system/*` 配置接口按职责分别落在对应业务模块中：规则配置归 `exceptiondetect/rule`，风险等级配置归 `warning`，异常类型配置归 `review`。
- `GET /api/log/operation/list` 归入 `statistics` 提供的审计查询能力，前端页面可挂在 `/system` 的日志页中展示。
- 如接口契约与实现冲突，以 `API接口设计文档.md` 为并发开发基准，再回写实现与测试。

## 3. 模块清单

### BE-01 用户与权限基础模块

- 模块名：用户与权限基础模块
- 范围：`auth`、`user`、`department`、`role`
- 依赖接口组：
  - `POST /api/auth/login`
  - `GET /api/user/list`
  - `POST /api/user/add`
  - `PUT /api/user/update`
  - `DELETE /api/user/{id}`
  - `GET /api/department/list`
  - `POST /api/department/add`
  - `PUT /api/department/update`
  - `DELETE /api/department/{id}`
  - `GET /api/role/list`
  - `POST /api/role/add`
  - `PUT /api/role/update`
  - `DELETE /api/role/{id}`
- 依赖前置模块：无
- 可并行边界：只负责登录态、用户主体、部门、角色、账号状态与权限边界；不承担设备、人脸、打卡、异常、预警逻辑。
- 完成标准：登录成功/失败可用；用户/部门/角色基础 CRUD 可用；管理员与员工权限边界明确。

### BE-02 设备基础资料模块

- 模块名：设备基础资料模块
- 范围：`device`
- 依赖接口组：
  - `GET /api/device/list`
  - `POST /api/device/add`
  - `PUT /api/device/update`
  - `PUT /api/device/status`
  - `DELETE /api/device/{deviceId}`
- 依赖前置模块：无业务硬依赖；鉴权接入时依赖 `BE-01`
- 可并行边界：只维护设备主数据与状态，不负责打卡、人脸、异常逻辑。
- 完成标准：设备新增、修改、状态切换、查询、删除可用；后续模块可稳定引用 `deviceId`。

### BE-03 人脸采集与识别模块

- 模块名：人脸采集与识别模块
- 范围：`face`
- 依赖接口组：
  - `POST /api/face/register`
  - `POST /api/face/verify`
- 依赖前置模块：`BE-01`
- 可并行边界：只输出人脸录入与验证结果、`faceScore` 和特征数据；不生成考勤记录与异常结论。
- 完成标准：可将人脸与员工绑定；录入与验证接口可用；验证结果可供打卡模块直接消费。

### BE-04 考勤打卡与记录模块

- 模块名：考勤打卡与记录模块
- 范围：`attendance`
- 依赖接口组：
  - `POST /api/attendance/checkin`
  - `GET /api/attendance/record/{userId}`
  - `GET /api/attendance/list`
  - `POST /api/attendance/repair`
- 依赖前置模块：`BE-01`、`BE-02`、`BE-03`
- 可并行边界：只负责原始打卡数据、记录查询与补卡申请；不承担异常分析、预警与复核逻辑。
- 完成标准：可生成完整打卡记录；个人与列表查询可用；补卡申请可提交；状态字段对下游模块可复用。

### BE-05 异常检测与智能分析模块

- 模块名：异常检测与智能分析模块
- 范围：`exceptiondetect`、`rule`
- 依赖接口组：
  - `POST /api/exception/rule-check`
  - `POST /api/exception/complex-check`
  - `GET /api/exception/list`
  - `GET /api/exception/{id}`
  - `GET /api/exception/{id}/decision-trace`
  - `GET /api/exception/{id}/analysis-brief`
  - `GET /api/rule/list`
  - `POST /api/rule/add`
  - `PUT /api/rule/update`
  - `PUT /api/rule/status`
- 依赖前置模块：`BE-04`
- 可并行边界：负责规则判定、复杂异常分析、结构化模型输出、判定依据、处理建议与决策追踪；不负责预警生成、复核提交、统计聚合。
- 完成标准：基础异常可判定并落库；复杂异常可产出结构化分析结果；规则配置接口可管理异常判定规则；异常详情可查看决策链路。

### BE-06 风险预警模块

- 模块名：风险预警模块
- 范围：`warning`
- 依赖接口组：
  - `GET /api/warning/list`
  - `GET /api/warning/{id}/advice`
  - `POST /api/warning/re-evaluate`
  - `GET /api/system/risk-level/list`
  - `PUT /api/system/risk-level/update`
- 依赖前置模块：`BE-05`
- 可并行边界：消费异常结果并生成智能预警摘要、优先级和处置建议，不处理人工复核结论。
- 完成标准：中高风险异常可生成预警记录；预警列表、预警建议和风险等级配置可用。

### BE-07 人工复核模块

- 模块名：人工复核模块
- 范围：`review`
- 依赖接口组：
  - `POST /api/review/submit`
  - `GET /api/review/{exceptionId}`
  - `GET /api/review/{exceptionId}/assistant`
  - `POST /api/review/feedback`
  - `GET /api/exception/{id}`
  - `GET /api/system/exception-type/list`
  - `PUT /api/system/exception-type/update`
- 依赖前置模块：`BE-05`
- 可并行边界：处理人工确认、驳回、AI 复核辅助、复核反馈归档与异常状态联动；不负责预警生成与统计。
- 完成标准：复核提交、AI 复核辅助与复核记录查询可用；异常处理状态可被正确更新；异常类型配置和反馈标签可维护。

### BE-08 统计分析与审计模块

- 模块名：统计分析与审计模块
- 范围：`statistics`
- 依赖接口组：
  - `GET /api/statistics/personal`
  - `GET /api/statistics/department`
  - `GET /api/statistics/exception-trend`
  - `GET /api/statistics/summary`
  - `GET /api/statistics/department-risk-brief`
  - `GET /api/statistics/export`
  - `GET /api/log/operation/list`
- 依赖前置模块：核心依赖 `BE-04`、`BE-05`；完整闭环依赖 `BE-06`、`BE-07`
- 可并行边界：负责聚合查询、趋势分析、智能摘要、风险画像、导出与审计日志查询，不回写业务流程。
- 完成标准：个人/部门统计、异常趋势、智能摘要、风险画像、导出、操作日志查询可用。

## 4. 串并行关系

### 必须串行

1. `BE-01 -> BE-03`：没有员工主体，无法做人脸绑定。
2. `BE-01 + BE-02 + BE-03 -> BE-04`：没有用户、设备、人脸结果，无法稳定生成打卡记录。
3. `BE-04 -> BE-05`：没有打卡记录，无法做异常判定与智能分析。
4. `BE-05 -> BE-06`：没有异常结果，无法生成预警。
5. `BE-05 -> BE-07`：没有异常结果，无法进行人工复核。
6. `BE-04 + BE-05 + BE-06 + BE-07 -> BE-08`：完整统计口径依赖主链路数据闭环。
7. `BE-07 -> BE-05`：复核反馈可反哺提示词和判定策略，但属于优化闭环，不阻塞主链路运行。

### 可以并行

1. `BE-01` 与 `BE-02` 可并行。
2. `BE-06` 与 `BE-07` 可在异常结果契约稳定后并行。
3. `BE-08` 可在核心表结构稳定后提前做读模型与统计口径。

## 5. 推荐并发波次

- 第一波：`BE-01 + BE-02`
- 第二波：`BE-03`
- 第三波：`BE-04`
- 第四波：`BE-05`
- 第五波：`BE-06 + BE-07 + BE-08`

## 6. 使用说明

- 新会话开始后，先读取本文件，再读取 `AGENTS.md` 与 API 主文档。
- 每次只认领一个模块编号，避免多人同时改同一模块目录。
- 如果某模块依赖接口尚未冻结，先补契约再并发开发实现。
- 若接口命名、字段或流程发生变化，必须同步更新本文件和 `API接口设计文档.md`。
