# 考勤异常检测与预警系统需求规格说明书

## 1. 引言

### 1.1 编写目的
本文档用于明确基于大模型的考勤异常检测与预警系统的功能需求、非功能需求、业务流程和验收指标，为系统设计、开发、测试和论文撰写提供依据。

### 1.2 项目背景
传统考勤系统多依赖固定规则和单一身份校验方式，难以准确识别代打卡、多设备异常打卡、多地点异常打卡和行为组合异常等复杂场景。为提升考勤管理的智能化水平，本系统在 `SpringBoot` 与 `Vue3` 技术栈基础上，引入大模型 `API`，对复杂异常行为进行综合判定与预警。

### 1.3 术语说明
- 基础异常：迟到、早退、缺勤、重复打卡等规则明确的异常
- 复杂异常：代打卡、多设备异常打卡、多地点异常打卡、行为组合异常等需多维分析的异常
- 风险等级：根据异常严重程度划分为低、中、高
- 人工复核：管理员对系统异常判定结果进行确认或驳回

## 2. 系统目标
- 实现员工、部门、角色与权限管理
- 实现基于人脸识别的考勤打卡
- 实现基础异常的自动判定
- 基于大模型增强复杂异常判定能力
- 实现异常预警、人工复核和统计分析
- 满足低成本、易部署、可扩展的中小企业应用需求

## 3. 用户角色

### 3.1 管理员
- 维护员工、部门、考勤规则
- 查看考勤、异常、预警和报表
- 进行人工复核
- 导出统计报表

### 3.2 员工
- 登录系统
- 录入人脸
- 日常打卡
- 查看个人考勤记录
- 提交补卡申请

## 4. 功能需求

### 4.1 用户与权限管理
- 员工信息增删改查
- 部门信息维护
- 角色分配
- 登录认证与权限控制
- 用户状态启停

### 4.2 人脸采集与打卡
- 摄像头采集
- 人脸检测与识别
- 打卡时间、设备、地点、IP 记录
- 打卡结果反馈

### 4.3 基础异常检测
- 迟到判定
- 早退判定
- 缺勤判定
- 重复打卡判定
- 非法时间打卡判定

### 4.4 复杂异常检测
- 代打卡判定
- 多设备异常打卡判定
- 多地点异常打卡判定
- 异常行为组合判定
- 持续性高风险模式识别

### 4.5 预警与复核
- 异常分级预警
- 站内提醒
- 异常详情查看
- 人工确认与驳回
- 复核记录归档

### 4.6 统计与报表
- 个人考勤统计
- 部门考勤统计
- 异常趋势分析
- 出勤率与异常率统计
- 报表导出

## 5. 非功能需求
- 响应时间不超过 3 秒
- 支持 30 人并发访问
- 异常识别准确率不低于 95%
- 支持普通 Windows 办公电脑部署
- 敏感数据加密存储
- 系统具备良好可维护性与可扩展性

## 6. 业务流程需求
- 员工登录 -> 打卡 -> 人脸校验 -> 记录生成
- 规则引擎初判 -> 风险特征提取 -> 大模型分析 -> 融合判定
- 异常落库 -> 预警推送 -> 管理员复核 -> 结果归档
- 统计汇总 -> 报表展示与导出

## 7. 约束条件
- 后端基于 `SpringBoot`
- 前端基于 `Vue3 + Element Plus`
- 使用 `MySQL + Redis`
- 支持摄像头采集与人脸比对
- 大模型能力通过外部 `API` 接入

## 8. 验收指标
- 基础打卡流程运行稳定
- 复杂异常检测可正常触发
- 预警流程完整可用
- 报表统计结果正确
- 数据安全控制有效
