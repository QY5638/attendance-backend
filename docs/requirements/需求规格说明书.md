# 考勤异常检测与预警系统需求规格说明书

## 1. 引言

### 1.1 编写目的
本文档用于明确基于大模型的考勤异常检测与预警系统的功能需求、非功能需求、业务流程和验收指标，为系统设计、开发、测试和论文撰写提供依据。

### 1.2 项目背景
传统考勤系统多依赖固定规则和单一身份校验方式，难以准确识别代打卡、多设备异常打卡、多地点异常打卡和行为组合异常等复杂场景。为提升考勤管理的智能化水平，本系统在 `SpringBoot` 与 `Vue3` 技术栈基础上，引入大模型 `API`，对复杂异常行为进行综合判定与预警。

### 1.3 术语说明
- 基础异常：迟到、早退、缺勤、重复打卡等规则明确的异常
- 复杂异常：代打卡、多设备异常打卡、多地点异常打卡、行为组合异常等需多维分析的异常
- 风险等级：根据异常严重程度划分为低、中、高
- 人工复核：管理员对系统异常判定结果进行确认或驳回

## 2. 系统目标
- 实现员工、部门、角色与权限管理
- 实现基于人脸识别的考勤打卡
- 实现基础异常的自动判定
- 基于大模型增强复杂异常判定能力
- 让大模型参与异常识别、预警生成、复核辅助和统计解释，形成全链路智能决策能力
- 建立提示词模板、模型调用日志和决策追踪机制，提升模型能力的工程可控性与可追溯性
- 建立人工复核反馈反哺提示词和判定策略的持续优化闭环
- 实现异常预警、人工复核和统计分析
- 满足低成本、易部署、可扩展的中小企业应用需求

## 3. 用户角色

### 3.1 管理员
- 维护员工、部门、考勤规则
- 查看考勤、异常、预警和报表
- 进行人工复核
- 导出统计报表

### 3.2 员工
- 登录系统
- 录入人脸
- 日常打卡
- 查看个人考勤记录
- 提交补卡申请

## 4. 功能需求

### 4.1 用户与权限管理
- 员工信息增删改查
- 部门信息维护
- 角色分配
- 登录认证与权限控制
- 用户状态启停

### 4.2 人脸采集与打卡
- 摄像头采集
- 人脸检测与识别
- 打卡时间、设备、地点、IP 记录
- 打卡结果反馈

### 4.3 基础异常检测
- 迟到判定
- 早退判定
- 缺勤判定（当日上班阈值后执行）
- 下班缺卡判定（次日 00:00 后结算前一日）
- 重复打卡判定
- 非法时间打卡判定（不将上班时间前的 `IN` 与下班时间后的 `OUT` 视为非法时间）

### 4.4 复杂异常检测
- 代打卡判定
- 多设备异常打卡判定
- 多地点异常打卡判定
- 异常行为组合判定
- 持续性高风险模式识别
- 异常判定依据生成
- 处理建议生成
- 相似风险案例归纳

### 4.5 预警与复核
- 异常分级预警
- 站内提醒
- 异常详情查看
- 人工确认与驳回
- 复核记录归档
- AI 预警摘要生成
- AI 预警优先级排序
- AI 复核建议与证据归纳
- 复核反馈标签采集与策略优化

### 4.6 统计与报表
- 个人考勤统计
- 部门考勤统计
- 异常趋势分析
- 出勤率与异常率统计
- 报表导出
- AI 趋势摘要生成
- 部门风险画像生成
- 周报与月报智能摘要生成
- 管理建议输出

### 4.7 模型策略与系统支撑
- 提示词模板管理
- 模型调用日志查询
- 决策追踪链路查看
- 结构化模型输出解析
- 模型失败降级与兜底处理

## 5. 非功能需求
- 响应时间不超过 3 秒
- 支持 30 人并发访问
- 异常识别准确率不低于 95%
- 支持普通 Windows 办公电脑部署
- 敏感数据加密存储
- 系统具备良好可维护性与可扩展性
- 模型调用前必须进行敏感信息脱敏
- 模型输出应满足结构化解析要求，避免仅返回自由文本
- 模型调用失败时系统需自动降级到规则判定或人工复核流程
- 模型调用、决策融合和反馈结果需可追踪、可审计

## 6. 业务流程需求
- 员工登录 -> 打卡 -> 人脸校验 -> 记录生成
- 规则引擎初判 -> 风险特征提取 -> 大模型分析 -> 融合判定
- 异常落库 -> 智能预警摘要生成 -> 管理员复核 -> 结果归档
- 人工复核结果 -> 反馈标签 -> 提示词和策略优化
- 统计汇总 -> 大模型智能解读 -> 报表展示与导出

## 7. 约束条件
- 后端基于 `SpringBoot`
- 前端基于 `Vue3 + Element Plus`
- 使用 `MySQL + Redis`
- 支持摄像头采集与人脸比对
- 大模型能力通过外部 `API` 接入
- 基础异常优先由规则引擎处理，复杂场景再触发大模型
- 模型能力需兼顾成本、稳定性和可解释性

## 8. 验收指标
- 基础打卡流程运行稳定
- 复杂异常检测可正常触发
- 预警流程完整可用
- 报表统计结果正确
- 数据安全控制有效
- AI 判定依据、风险解释和处理建议可展示
- AI 预警摘要、AI 复核建议和 AI 统计摘要可正常生成
- 模型调用日志、决策追踪链路和反馈回流机制可运行
