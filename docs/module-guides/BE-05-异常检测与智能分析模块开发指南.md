# BE-05 异常检测与智能分析模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-05` 的独立开发。
- 开发前先读取后端并发清单、API 文档异常检测章节和 `BE-04` 输出记录字段。

## 2. 模块目标

- 实现基础规则异常判定、复杂异常智能分析和决策追踪输出。

## 3. 模块边界

- 处理：`exceptiondetect` 和 `rule` 子域。
- 输出异常结果、结构化模型结果、判定依据、处理建议和决策链路。
- 不生成预警，不提交复核。

## 4. 前置依赖

- `BE-04`

## 5. 固定接口清单

1. `POST /api/exception/rule-check`
2. `POST /api/exception/complex-check`
3. `GET /api/exception/list`
4. `GET /api/exception/{id}`
5. `GET /api/exception/{id}/decision-trace`
6. `GET /api/exception/{id}/analysis-brief`
7. `GET /api/rule/list`
8. `POST /api/rule/add`
9. `PUT /api/rule/update`
10. `PUT /api/rule/status`

## 6. 固定数据表与字段约定

- 表：`rule`、`attendanceException`、`exceptionAnalysis`、`promptTemplate`、`modelCallLog`、`decisionTrace`。
- 输出重点字段：`type`、`riskLevel`、`sourceType`、`processStatus`、`modelConclusion`、`reasonSummary`、`actionSuggestion`、`confidenceScore`。

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/exceptiondetect/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 实现规则配置查询、新增、修改、状态切换。
2. 实现迟到、早退、缺勤、重复打卡、非法时间等基础规则判定。
3. 提取复杂异常分析所需的风险特征。
4. 根据场景选择提示词模板并组织结构化输入。
5. 实现复杂异常分析入口和结果落库。
6. 解析结构化模型输出，生成结论、依据、建议和置信度。
7. 记录模型调用日志与决策追踪链路。
8. 实现异常列表、异常详情、决策链和分析摘要查询。
9. 补齐规则管理、异常判定、模型输出解析和决策追踪测试。

## 9. 最小验证命令

```bash
mvn -Dtest=ExceptionControllerTest test
```

## 10. 完成标准

- 基础异常可自动落库。
- 复杂异常分析接口可产出结构化结果。
- 异常详情可展示 AI 摘要与决策链。
- 预警和复核模块可直接消费异常结果。

## 11. 禁止改动项

- 不在本模块生成预警。
- 不在本模块处理人工复核。
- 不在本模块直接输出统计报表。

## 12. 文档同步要求

- 改规则接口、异常字段或状态口径时，同步 API 与测试文档。
- 改表结构时同步 SQL 与数据库文档。
