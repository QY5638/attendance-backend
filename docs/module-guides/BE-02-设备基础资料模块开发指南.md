# BE-02 设备基础资料模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-02` 的独立开发。
- 开发前先读取后端并发清单和 API 文档设备章节。

## 2. 模块目标

- 完成设备主数据管理和设备状态维护。
- 为打卡模块提供稳定的设备基础资料。

## 3. 模块边界

- 只处理：`device`。
- 不处理：人脸验证、打卡、异常分析、预警和复核。

## 4. 前置依赖

- 鉴权接入依赖 `BE-01`。

## 5. 固定接口清单

1. `GET /api/device/list`
2. `POST /api/device/add`
3. `PUT /api/device/update`
4. `PUT /api/device/status`
5. `DELETE /api/device/{deviceId}`

## 6. 固定数据表与字段约定

- 主表：`device`。
- 数据库主键当前为 `id`。
- 接口层统一对外字段为 `deviceId`。
- 状态字段用于设备启停控制。

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/device/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 明确 `id -> deviceId` 的实体映射和 VO 输出。
2. 实现设备列表查询。
3. 实现设备新增。
4. 实现设备修改。
5. 实现设备状态切换。
6. 实现设备删除。
7. 校验设备编号唯一性和基础字段完整性。
8. 补齐 `DeviceManagementIntegrationTest`，并检查所有输出字段都使用 `deviceId`。

## 9. 最小验证命令

```bash
mvn "-Dtest=DeviceManagementIntegrationTest,AuthSecurityIntegrationTest,ModuleSkeletonBeansTest" test
mvn -DskipTests compile
```

## 10. 完成标准

- 设备 CRUD 可用。
- 状态切换可用。
- 所有接口统一输出 `deviceId`。
- 删除已被打卡记录引用的设备时返回业务错误。

## 11. 禁止改动项

- 不在本模块生成考勤记录。
- 不扩展异常检测逻辑。

## 12. 文档同步要求

- 若设备字段、状态或接口变化，必须同步 API、数据库和测试文档。
