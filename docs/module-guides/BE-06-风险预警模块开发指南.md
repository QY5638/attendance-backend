# BE-06 风险预警模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-06` 的独立开发。
- 开发前先读取后端并发清单、API 文档预警章节以及 `BE-05` 的异常输出约定。

## 2. 模块目标

- 根据异常结果生成智能预警，并维护风险等级配置与预警建议能力。

## 3. 模块边界

- 只处理：`warning`。
- 负责预警摘要、优先级和处置建议生成。
- 不处理人工复核。

## 4. 前置依赖

- `BE-05`

## 5. 固定接口清单

1. `GET /api/warning/list`
2. `GET /api/warning/{id}/advice`
3. `POST /api/warning/re-evaluate`
4. `GET /api/system/risk-level/list`
5. `PUT /api/system/risk-level/update`

## 6. 固定数据表与字段约定

- 主表：`warningRecord`。
- 输出重点字段：`exceptionId`、`type`、`level`、`status`、`priorityScore`、`aiSummary`、`disposeSuggestion`、`sendTime`。
- 风险等级配置承载方式必须与当前 SQL 和文档一致。

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/warning/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 明确风险等级配置结构。
2. 实现风险等级查询和更新。
3. 实现异常到预警的生成规则。
4. 生成 AI 预警摘要、优先级和处置建议。
5. 实现预警列表查询与预警建议查询。
6. 实现预警重评估接口。
7. 固定预警状态流转字段。
8. 补齐预警生成、查询、重评估、配置更新测试。

## 9. 最小验证命令

```bash
mvn -Dtest=WarningControllerTest test
```

## 10. 完成标准

- 中高风险异常能生成预警。
- 预警列表可查。
- 预警建议和优先级可查。
- 风险等级配置可维护。

## 11. 禁止改动项

- 不在本模块处理复核记录。
- 不扩展消息通知系统。
- 不在本模块直接修改异常分析逻辑。

## 12. 文档同步要求

- 若风险等级字段、状态或接口变化，需同步 API、数据库和测试文档。
