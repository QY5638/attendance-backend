# BE-08 统计分析与审计模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-08` 的独立开发。
- 开发前先读取后端并发清单、API 文档统计章节及 `BE-04` 到 `BE-07` 的输出约定。

## 2. 模块目标

- 实现个人统计、部门统计、异常趋势、智能摘要、风险画像、导出和操作日志查询。
- 统计口径必须串通打卡、异常、预警、复核和审计链路，不做只覆盖 `BE-04/05` 的半成品。

## 3. 模块边界

- 只做读模型、聚合和导出。
- 不回写业务主流程。
- 负责统计解释和管理建议生成。
- 为支撑审计查询，可最小补齐关键成功路径的 `operationLog` 写入，但不改异常、预警、复核业务判定逻辑。

## 4. 前置依赖

- 核心依赖：`BE-04`、`BE-05`
- 完整闭环依赖：`BE-06`、`BE-07`

## 5. 固定接口清单

1. `GET /api/statistics/personal`
2. `GET /api/statistics/department`
3. `GET /api/statistics/exception-trend`
4. `GET /api/statistics/summary`
5. `GET /api/statistics/department-risk-brief`
6. `GET /api/statistics/export`
7. `GET /api/log/operation/list`

接口冻结约束：
- `/api/statistics/personal` 按当前会话推导统计用户，前端不传 `userId`
- `/api/statistics/department` 仅 `ADMIN` 可访问，默认返回全局所有部门汇总
- `/api/statistics/summary` 同一路径支持 `ADMIN` 和 `EMPLOYEE`，分别返回全局/个人摘要
- `/api/statistics/export` 固定返回 `CSV` 文件下载，`Content-Type` 为 `text/csv;charset=UTF-8`

## 6. 固定数据表与字段约定

- 统计来源：`attendanceRecord`、`attendanceException`、`exceptionAnalysis`、`warningRecord`、`reviewRecord`、`user`、`department`。
- 审计表：`operationLog`。
- 审计完备性评估：`modelCallLog`、`decisionTrace`。
- 日志查询接口归属本模块提供。
- 输出重点还包括 AI 趋势摘要、风险画像和管理建议。
- 时间锚点统一使用 `attendanceRecord.checkTime`。
- 完整闭环定义：同一异常同时存在 `warningRecord` 与 `reviewRecord`。
- 核心指标至少覆盖：
  - 打卡总量/异常量
  - 异常分析完成量
  - 异常类型与风险等级分布
  - 预警数量与状态分布
  - 复核结果分布与闭环数量
  - 操作日志查询

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/statistics/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 固定统计口径和查询维度。
2. 实现个人统计，覆盖打卡、异常、预警、复核闭环字段。
3. 实现部门统计，默认输出全局所有部门汇总口径。
4. 实现异常趋势统计，支持 `DAY/WEEK/MONTH` 周期聚合。
5. 生成统计摘要、风险提示和管理建议，并纳入审计缺口评估；管理员看全局，员工看个人。
6. 生成部门风险画像摘要与风险评分。
7. 实现 `CSV` 导出接口，沿用项目现有依赖，不引入重依赖，并以文件下载方式返回。
8. 实现操作日志查询接口。
9. 最小补齐 `operationLog` 成功路径写入：登录、打卡、补卡申请、预警重评估、复核提交、复核反馈。
10. 补齐统计、摘要、导出、日志和审计写链测试。

## 9. 最小验证命令

```bash
mvn -Dtest=StatisticsControllerTest test
```

## 10. 完成标准

- 统计、趋势、导出和日志查询都可用。
- 智能摘要和风险画像可用。
- 前端 dashboard、statistics、system/log 页面可直接接入。
- `BE-04 -> BE-05 -> BE-06 -> BE-07 -> BE-08` 的完整闭环统计口径可跑通。
- `modelCallLog`、`decisionTrace` 已纳入摘要层面的审计完备性评估。

## 11. 禁止改动项

- 不在本模块回写异常、预警、复核状态。
- 不把统计逻辑散落到其他模块。
- 不在本模块重新定义业务状态流转。

## 12. 文档同步要求

- 若统计口径、导出字段或日志字段变化，需同步 API 与测试文档。
