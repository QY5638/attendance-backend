# BE-03 人脸采集与识别模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-03` 的独立开发。
- 开发前先读取后端并发清单、API 文档人脸接口章节和 `BE-01` 的用户主体约定。

## 2. 模块目标

- 实现员工人脸录入、特征保存和身份验证能力。

## 3. 模块边界

- 只处理：`face`。
- 输出验证结果和 `faceScore`。
- 不生成考勤记录。

## 4. 前置依赖

- `BE-01`

## 5. 固定接口清单

1. `POST /api/face/register`
2. `POST /api/face/verify`

## 6. 固定数据表与字段约定

- 主表：`faceFeature`。
- 前端只消费验证结果、分数和必要提示。
- 敏感原始特征数据不直接对前端暴露。

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/face/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 定义人脸特征实体、绑定关系和 DTO/VO。
2. 实现人脸录入接口。
3. 实现人脸验证接口。
4. 输出验证分数、是否通过、人脸是否已注册等结果。
5. 预留第三方识别服务封装点。
6. 处理录入失败、未注册、验证失败等异常情况。
7. 补齐录入与验证测试。

## 9. 最小验证命令

```bash
mvn -Dtest=FaceControllerTest test
```

## 10. 完成标准

- 人脸录入可用。
- 人脸验证可用。
- 打卡模块可直接消费验证结果。

## 11. 禁止改动项

- 不在本模块落打卡记录。
- 不直接返回原始特征内容。

## 12. 文档同步要求

- 若接口入参或出参调整，需同步 API 和测试文档。
- 若表结构变化，需同步 SQL 与数据库文档。
