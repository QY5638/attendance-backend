# BE-07 人工复核模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-07` 的独立开发。
- 开发前先读取后端并发清单、API 文档复核章节和 `BE-05` 异常详情契约。

## 2. 模块目标

- 实现异常人工复核、AI 复核辅助、反馈标签采集和异常状态联动。

## 3. 模块边界

- 只处理：`review`。
- 不生成预警。
- 不做统计聚合。
- 负责复核建议和反馈学习入口。

## 4. 前置依赖

- `BE-05`

## 5. 固定接口清单

1. `POST /api/review/submit`
2. `GET /api/review/{exceptionId}`
3. `GET /api/review/{exceptionId}/assistant`
4. `POST /api/review/feedback`
5. `GET /api/exception/{id}`
6. `GET /api/system/exception-type/list`
7. `PUT /api/system/exception-type/update`

## 6. 固定数据表与字段约定

- 主表：`reviewRecord`。
- 关联表：`attendanceException`。
- 输出重点字段：`exceptionId`、`reviewResult`、`reviewComment`、`aiReviewSuggestion`、`similarCaseSummary`、`feedbackTag`。

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/review/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 实现异常类型配置查询和更新。
2. 实现 AI 复核辅助信息查询。
3. 实现复核提交。
4. 保存复核记录并回写异常状态。
5. 实现复核反馈标签和策略反馈提交。
6. 实现复核记录查询。
7. 补齐辅助查询、提交成功、提交失败、状态联动、历史查询测试。

## 9. 最小验证命令

```bash
mvn -Dtest=ReviewControllerTest test
```

## 10. 完成标准

- 复核提交可用。
- AI 复核辅助信息可查。
- 复核记录可查。
- 异常状态联动正确。
- 反馈标签可沉淀并供后续策略优化使用。

## 11. 禁止改动项

- 不在本模块生成预警。
- 不在本模块实现统计报表。
- 不在本模块直接修改提示词模板内容。

## 12. 文档同步要求

- 改复核字段、异常类型字段或接口时，同步 API 和测试文档。
- 改表结构时同步 SQL 与数据库文档。
