# BE-04 考勤打卡与记录模块开发指南

## 1. 文档说明

- 本文件用于指导 `BE-04` 的独立开发。
- 开发前先读取后端并发清单、API 文档打卡与记录章节，以及 `BE-01/02/03` 的输出约定。

## 2. 模块目标

- 实现上下班打卡、个人记录查询、管理端记录列表和补卡申请。

## 3. 模块边界

- 只处理：`attendance`。
- 生成原始考勤记录。
- 不输出异常判定结论。

## 4. 前置依赖

- `BE-01`
- `BE-02`
- `BE-03`

## 5. 固定接口清单

1. `POST /api/attendance/checkin`
2. `GET /api/attendance/record/{userId}`
3. `GET /api/attendance/list`
4. `POST /api/attendance/repair`

## 6. 固定数据表与字段约定

- 主表：`attendanceRecord`。
- 打卡请求重点字段：`userId`、`checkType`、`deviceId`、`ipAddr`、`location`。
- 记录状态需供异常模块复用。

## 7. 代码目录范围

- `src/main/java/com/quyong/attendance/module/attendance/`
- `src/test/java/com/quyong/attendance/`

## 8. 详细开发步骤

1. 定义打卡、查询、补卡相关 DTO/VO。
2. 接入用户、设备和人脸验证前置结果。
3. 实现上下班打卡和原始记录落库。
4. 实现个人记录查询。
5. 实现管理端分页列表查询。
6. 实现补卡申请。
7. 固定记录状态字段。
8. 补齐打卡、记录、补卡测试。

## 9. 最小验证命令

```bash
mvn -Dtest=AttendanceControllerTest test
```

## 10. 完成标准

- 打卡可提交。
- 记录可查询。
- 补卡可申请。
- 数据可被异常检测模块直接消费。

## 11. 禁止改动项

- 不在本模块生成异常、预警、复核结果。

## 12. 文档同步要求

- 改接口同步 API 文档。
- 改记录字段或表结构同步 SQL、数据库和测试文档。
