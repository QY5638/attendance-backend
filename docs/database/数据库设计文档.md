# 考勤异常检测与预警系统数据库设计文档

## 1. 数据库设计目标
数据库用于支撑员工管理、人脸特征管理、考勤记录、异常检测、预警处理、人工复核和统计分析等核心业务。

## 2. 数据库选型
- 数据库名称：`system`
- 关系型数据库：`MySQL`
- 缓存数据库：`Redis`

## 4. 表关系设计
- 一个部门对应多个员工
- 一个角色对应多个员工
- 一个员工对应多条打卡记录
- 一个员工对应多条补卡申请
- 一个员工对应一条或多条人脸特征记录
- 一个员工可发起多条人脸重录申请，并由管理员审批
- 一台设备可对应多条打卡记录
- 一条打卡记录可关联零条或多条补卡申请处理记录
- 一条打卡记录可对应一条或多条异常记录
- 一条异常记录可关联智能分析、预警和复核记录
- 一条异常记录可关联多条预警交互记录
- 一条预警记录可关联多条站内通知与交互记录
- 一条异常记录可关联一条或多条决策追踪记录
- 一条异常记录的 `type` 字段可引用一条异常类型配置
- 一条异常记录的 `riskLevel` 字段可引用一条风险等级配置
- 一个提示词模板可关联多条模型调用日志
- 一条复核记录可沉淀反馈标签与策略反馈结果

## 5. 索引设计
建议建立如下索引：
- `attendanceRecord(userId, checkTime)`
- `attendanceRecord(deviceId, checkTime)`
- `attendanceRepair(userId, checkTime)`
- `attendanceRepair(userId, checkType, checkTime, status)` 唯一约束
- `attendanceException(userId, createTime)`
- `warningRecord(exceptionId)`
- `warningInteractionRecord(warningId, createTime)`
- `notificationRecord(recipientUserId, readStatus, createTime)`
- `notificationRecord(businessType, businessId, category)`
- `faceRegisterApproval(userId, createTime)`
- `faceRegisterApproval(status, createTime)`
- `reviewRecord(exceptionId)`
- `exceptionType(code)` 唯一索引
- `exceptionType(status)`
- `riskLevel(code)` 唯一索引
- `riskLevel(status)`
- `modelCallLog(businessType, businessId)`
- `modelCallLog(promptTemplateId)`
- `decisionTrace(businessType, businessId)`
- `promptTemplate(sceneType, status)`

## 6. 安全与性能设计
- 人脸特征字段加密存储
- 敏感字段访问控制
- 使用 `Redis` 维护访问令牌、登录限流状态与活体证明等临时数据
- 高频查询字段建立索引
- 模型输入在日志中采用摘要化保存，避免直接落敏感原文
- 决策追踪链路需保留规则结果、模型结果和最终结果，便于审计
