# 考勤异常检测与预警系统测试用例文档

## 1. 文档说明

### 1.1 编写目的
本文档用于描述考勤异常检测与预警系统的测试目标、测试范围、测试环境、测试策略及核心测试用例，为系统联调、验收和毕业设计测试章节撰写提供依据。

### 1.2 测试目标
- 验证系统各功能模块运行正确
- 验证基础考勤异常判定逻辑准确性
- 验证复杂考勤异常识别流程可用性
- 验证预警、人工复核和报表功能完整性
- 验证系统性能、安全性和可用性满足设计要求

## 2. 测试环境

### 2.1 硬件环境
- 普通 Windows 办公电脑
- 摄像头设备
- 本地网络环境

### 2.2 软件环境
- 操作系统：Windows 10 或以上
- JDK：1.8 及以上
- 前端：Vue3 + Element Plus
- 后端：SpringBoot
- 数据库：MySQL 8.x，使用 `system` 库
- 缓存：Redis 6.x
- 浏览器：Chrome
- 接口测试工具：Postman

测试相关核心表：
- `user`、`department`、`role`
- `device`、`faceFeature`、`rule`
- `attendanceRecord`、`attendanceException`
- `exceptionAnalysis`、`warningRecord`、`reviewRecord`、`operationLog`

测试数据建议通过 `D:\Graduation project\backend\sql\seed\测试数据插入SQL.sql` 导入。

## 3. 测试范围
- 用户与权限管理
- 人脸采集与打卡
- 基础异常判定
- 复杂异常智能分析
- 预警与人工复核
- 统计分析与报表
- 安全性与性能

### 5.2 异常场景输入示例
以下示例中，示例一至示例三可在导入当前种子 SQL 后直接复现；示例四可结合现有原始记录验证，若需独立验证也可补充构造。
- 示例一：迟到。员工 `1002` 于 `09:16` 使用 `DEV-002` 打卡，预期判定为迟到。
- 示例二：重复打卡。员工 `1001` 在 `08:58` 和 `09:00` 通过 `DEV-001` 连续打卡，预期判定为重复打卡。
- 示例三：疑似代打卡。员工 `1002` 于 `08:58:30` 使用异常设备 `DEV-009` 打卡，人脸相似度 `82.4`，预期触发复杂异常分析。
- 示例四：多设备异常。当前种子 SQL 已包含用户 `1002` 的跨设备原始记录，但未单独提供多设备异常结果；如需独立验证，仍可为同一员工补充短时间内跨设备打卡记录后再验证多设备异常分析。
